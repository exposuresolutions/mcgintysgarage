?? Key Learnings from Projects

  • Phone Numbers First: Always confirm the correct phone number BEFORE building. We had to update across 10+ files on Lavelle's project.
  • GHL vs GitHub: Clients don't understand the difference. Clear explanation needed: GitHub = code storage, GHL = live hosting.
  • Navigation Links: Must use full GHL URLs (https://subdomain.exposuresolutions.me/page), not relative paths like /page.html
  • Emojis: Can get corrupted when copy/pasting. Use sparingly or via Unicode.
  • Feedback Early: Get client approval on name/domain/branding BEFORE building full site. Use survey!
  • Mobile Action Bar: Essential for mobile users - always include Call + WhatsApp sticky buttons.
  • Partner Services: Clearly distinguish "our services" vs "partner referrals" to avoid confusion.
  • Domain Search: Include live domain availability check in strategy page so clients can decide on the spot.
